# Purpose

This checklist defines the minimum design requirements for software products before implementation, release, or production approval.

It applies to

- SaaS Platforms
- AI Applications
- Web Applications
- Mobile Applications
- Enterprise Software
- Developer Tools
- Dashboards
- Internal Platforms
- APIs with User Interfaces

Design is not decoration.

Design is the systematic process of solving user problems through clarity, usability, accessibility, consistency, and visual communication.

A beautiful interface that confuses users is poor design.

# Definition of Done

Design review is complete when

- User needs, business objectives, and product goals are clearly aligned through intuitive and accessible design solutions.
- Information architecture, navigation, layout, and interaction patterns minimize cognitive load while enabling users to complete critical tasks efficiently.
- Visual design consistently applies reusable components, typography, spacing, color systems, and branding across the entire product.
- Accessibility standards ensure that users with diverse abilities can successfully understand, navigate, and operate the interface.
- Forms, feedback mechanisms, loading states, error handling, and recovery flows support confident and frustration-free user interactions.
- Responsive behavior provides a high-quality experience across mobile, tablet, desktop, and large-screen environments.
- AI-powered experiences, where applicable, clearly communicate capabilities, limitations, confidence, and user control.
- Usability testing, cross-functional reviews, analytics, and user feedback validate that design decisions improve both customer satisfaction and business outcomes.
- Documentation captures design systems, interaction guidelines, accessibility requirements, reusable patterns, and implementation standards.
- The complete experience demonstrates clarity, consistency, trustworthiness, scalability, and long-term maintainability while making the product feel intuitive from the very first interaction.

Exceptional design is rarely noticed.

Users simply accomplish their goals with confidence, understand every interaction without explanation, recover naturally from mistakes, trust the product instinctively, and leave with the feeling that the software was designed specifically for them.
